                     AURA BUDGET — SYSTEM ARCHITECTURE
================================================================================

PROJECT OVERVIEW
----------------
Aura Budget is an AI-powered personal finance management web application that
combines transaction tracking, budget planning, investment tracking, goal
management, and collaborative shared budgets — all enhanced by LLM-powered
insights and ML-based spending forecasts.

FRONTEND LAYER
================================================================================

Technology Stack:
  • React 18          — UI framework
  • Vite              — Dev server & bundler
  • Axios             — HTTP client for API calls
  • Socket.IO Client  — Real-time events (room updates, join requests)

  • jsPDF             — PDF export for transaction reports

Application Screens (tabs in App.jsx):
  ┌──────────────────────────────────────────────────────────┐
  │  Overview       — Dashboard summary (income vs expenses) │
  │  Transactions   — Full transaction history + add/delete  │
  │  Budgets        — Category budget limits + AI Builder    │
  │  Subscriptions  — Recurring subscriptions tracker        │
  │  Goals          — Savings goals with AI insights         │
  │  Investments    — Portfolio tracker (stocks/crypto)      │
  │  Collaboration  — Shared Budget Rooms (multi-user)       │
  │  Diary          — "Describe Your Day" AI parser          │
  │  Forecast       — ML-powered spending forecast charts    │
  │  Settings       — Theme, currency, onboarding, reset     │
  └──────────────────────────────────────────────────────────┘

State Management:
  • All state managed via React useState hooks (no Redux)
  • JWT stored in localStorage; passed as Bearer token on every API request

REST API ENDPOINTS
================================================================================

  Auth & User
  ─────────────────────────────────────────────────────────────
  POST   /api/auth/register          Register a new user
  POST   /api/auth/login             Login, returns JWT
  POST   /api/user/onboarding        Save onboarding responses
  POST   /api/user/reset             Reset account data (requires password)

  Transactions
  ─────────────────────────────────────────────────────────────
  GET    /api/transactions           Get all transactions for user
  POST   /api/transactions           Create new transaction
  DELETE /api/transactions/:id       Delete a transaction

  Budgets
  ─────────────────────────────────────────────────────────────
  GET    /api/budgets                Get personal budgets
  POST   /api/budgets                Create or update budget for category
  DELETE /api/budgets/:id            Delete a budget

  Subscriptions
  ─────────────────────────────────────────────────────────────
  GET    /api/subscriptions          Get all subscriptions
  POST   /api/subscriptions          Add new subscription
  DELETE /api/subscriptions/:id      Remove subscription

  Goals
  ─────────────────────────────────────────────────────────────
  GET    /api/goals                  Get all goals
  POST   /api/goals                  Create new goal
  PUT    /api/goals/:id              Update goal (contribute amount)
  DELETE /api/goals/:id              Delete a goal

  Investments
  ─────────────────────────────────────────────────────────────
  GET    /api/investments            Get portfolio
  POST   /api/investments            Add investment asset
  PUT    /api/investments/:id        Update asset (price/shares)
  DELETE /api/investments/:id        Remove asset

  Stock Data (Alpha Vantage Integration)
  ─────────────────────────────────────────────────────────────
  GET    /api/stocks/search          Search for stock symbols
  GET    /api/stocks/price/:symbol   Get current price for a symbol

  Collaboration Rooms
  ─────────────────────────────────────────────────────────────
  GET    /api/rooms                  Get all rooms for user
  POST   /api/rooms                  Create a new budget room
  POST   /api/rooms/join             Request to join room by code
  POST   /api/rooms/:id/request/:rid Accept or reject join request
  POST   /api/rooms/:id/settle       Record a settlement between members
  DELETE /api/rooms/:id              Delete a room (admin only)
  GET    /api/rooms/:id/transactions Room transactions
  POST   /api/rooms/:id/budgets      Set room-level budget
  POST   /api/rooms/:id/comments     Post comment to room
  DELETE /api/rooms/:id/members/:uid Remove a member from room

  Diary / Journal
  ─────────────────────────────────────────────────────────────
  GET    /api/diary                  Get journal entries
  POST   /api/diary                  Save a new journal entry
  DELETE /api/diary/:id              Delete a journal entry

  AI Features (LLM)
  ─────────────────────────────────────────────────────────────
  POST   /api/insights               AI financial insights (Groq/LLaMA 3.3)
  POST   /api/ai/suggest-budget      AI budget generation from description
  POST   /api/ai/parse-day           Parse diary text → transactions/subs/goals
  POST   /api/ai/stock-suggestions   AI investment ideas (Groq/LLaMA 3.3)
  POST   /api/ai/transcribe          Voice-to-text transcription (Whisper via OpenAI)

  ML Forecasting
  ─────────────────────────────────────────────────────────────
  POST   /api/forecast               Invoke Python ML model → spending forecast

================================================================================
AI & MACHINE LEARNING LAYER
================================================================================

  LLM APIs Used:
    • Groq (LLaMA 3.3 70B Versatile)
        - /api/insights        — Personalized financial insights
        - /api/ai/suggest-budget — AI-generated budget plans
        - /api/ai/parse-day    — Natural language → structured transactions
        - /api/ai/stock-suggestions — Investment recommendations

    • Google Gemini API (@google/generative-ai)
        - Secondary LLM, configured and available in server.js

    • OpenAI Whisper (via OpenAI SDK)
        - /api/ai/transcribe  — Voice recording → text transcription

  ML Models (Python / scikit-learn — in /backend/model/):
    • expense_category_rf.pkl      — Classifies expense category from text
    • expense_amount_rf.pkl        — Predicts expected expense amount
    • spending_forecaster.pkl      — Time-series spending forecast
    • Training script: /backend/ml/train_rf.py (uses Kaggle personal finance data)
    • Inference script: /backend/forecast.py (called by Express via child process)

  External Data APIs:
    • Alpha Vantage API — Stock symbol search & real-time price lookup
    • Open Exchange Rates — Currency conversion for multi-currency support

================================================================================
REAL-TIME LAYER (Socket.IO)
================================================================================

  Events emitted by server → client:
    • room_update        — Room state changed (member joined, transaction added)
    • new_comment        — New comment posted in a room
    • new_transaction    — New transaction added to a room
    • new_join_request   — Someone requested to join a room
    • room_deleted       — Room was deleted by admin
    • request_processed  — Join request accepted or rejected

  Events emitted by client → server:
    • join_user_room     — Subscribe to user's personal notification channel
    • join_room          — Subscribe to a specific budget room's events
    • leave_room         — Unsubscribe from a room

================================================================================
AUTHENTICATION & SECURITY
================================================================================

  • Algorithm    : JWT (jsonwebtoken), signed with JWT_SECRET from .env
  • Password     : bcryptjs (salt rounds = 8)
  • Auth Flow    : Register → hashed password stored → login → JWT issued
                  → All protected routes use `auth` middleware to verify token
  • Token Storage: localStorage in browser (sent as Authorization: Bearer <token>)

================================================================================
DATA FLOW EXAMPLES
================================================================================

  1. User logs in:
     Browser → POST /api/auth/login → Express validates password (bcrypt)
             → issues JWT → stored in localStorage → all future requests
               attach token as Bearer header

  2. Describe Your Day (AI Diary):
     User speaks → MediaRecorder → audio blob → POST /api/ai/transcribe
     (Whisper) → text returned → POST /api/ai/parse-day (Groq/LLaMA) →
     JSON: { transactions[], subscriptions[], goals[] } → each item
     posted to respective API endpoint → UI refreshes

  3. AI Budget Builder:
     User types financial goal description → POST /api/ai/suggest-budget
     → Groq fetches user's real spending, income, subscriptions →
     LLaMA generates budget JSON → displayed in UI → user applies it
     → POST /api/budgets for each category

  4. Spending Forecast:
     POST /api/forecast → Express spawns child_process running forecast.py
     → Python loads spending_forecaster.pkl → predicts next N days →
     returns JSON → rendered as AreaChart in frontend

  5. Collaboration Room (real-time):
     User A adds transaction to Room → POST /api/transactions with
     collaborationId → server saves + emits 'new_transaction' via Socket.IO
     → User B (in same room) receives event → fetchRoomData() called →
     UI updates instantly without refresh
